# LLM 답변 판단 기준

## 1. 답변 생성 여부 (should_respond)

### ✅ 답변 생성 (should_respond: true)

LLM이 다음 조건을 **모두** 만족하면 답변 생성:

#### 조건 1: 명확한 분류 가능
- 11개 카테고리 중 하나로 명확히 분류됨
- 신뢰도 레벨: very_high 또는 high

#### 조건 2: Agent가 처리 가능
다음 중 **하나라도** 해당:

**A. FAQ로 답변 가능 (Prompt 직접)**
- 정적인 정보 (강의 목록, 제출 규정, 가이드)
- 일반적인 트러블슈팅 (영상 재생, 브라우저 호환성)
- 건의 접수 (감사 메시지)

**B. 과거 사례 + Tool로 답변 가능 (Tool + RAG)**
- 유사한 과거 사례 존재
- 코드 분석/디버깅 가능
- 과제 개발 가이드 제공 가능

#### 조건 3: 시스템 조치 불필요
다음 **모두** 해당하지 않음:
- ❌ 개인 계정 DB 직접 수정
- ❌ 시스템 백엔드 조치
- ❌ 관리자 권한 필요

### 예시

```
✅ 답변 생성 O
- "최종과제 여러 번 제출 가능한가요?" → FAQ 답변 가능
- "TypeError 발생합니다" → 과거 사례 + 디버깅 가능
- "파이썬 강의 어디 있나요?" → FAQ 답변 가능

❌ 답변 생성 X
- "인증 버튼 비활성화" → 계정 DB 조작 필요
- "console 접근 안됨" → 시스템 백엔드 문제
```

---

### ❌ 답변 안함 (should_respond: false)

LLM이 다음 중 **하나라도** 해당하면 답변 안함:

#### 이유 1: 시스템 조치 필요
- 개인 계정 권한 변경
- 플랫폼 백엔드 에러
- 데이터베이스 직접 조작

#### 이유 2: 정보 부족
- 개인별 상황 확인 필요
- 시스템 로그 분석 필요
- 추가 정보 없이 답변 불가

#### 이유 3: 위험성
- 잘못된 답변 시 사용자에게 악영향
- 보안/개인정보 관련
- 중요한 정책 결정 필요

## 2. 미분류 (UNCATEGORIZED)

### 발생 조건

LLM이 다음 중 **하나라도** 해당하면 미분류:

#### 조건 1: 신뢰도 부족
```python
confidence_level == "low"  # 신뢰도 레벨: 낮음
```

**예시**:
- 애매한 질문: "이거 어떻게 해요?"
- 여러 주제 혼재: "강의도 보고 과제도 제출하고 싶은데..."
- 불명확한 문의: "안돼요"

#### 조건 2: 11개 카테고리에 맞지 않음

**기존 카테고리**:
1. ACCOUNT_ACTION_REQUIRED
2. PLATFORM_SYSTEM_ERROR
3. COURSE_INFO
4. SUBMISSION_POLICY
5. SERVICE_GUIDE
6. FEATURE_REQUEST
7. PLATFORM_API_USAGE
8. VIDEO_PLAYBACK_ERROR
9. ASSIGNMENT_DEVELOPMENT
10. CODE_LOGIC_ERROR
11. UNCATEGORIZED ← 기본값

**예시**:
- 완전히 새로운 유형
- 카테고리 경계가 애매함
- 복합적인 문의

#### 조건 3: LLM 분석 실패
- API 오류
- JSON 파싱 실패
- 타임아웃

## 3. LLM 판단 Flow

```
┌─────────────────┐
│  문의 입력       │
└────────┬────────┘
         │
         ▼
┌─────────────────────────┐
│ LLM 분석                 │
│ 1. 내용 이해             │
│ 2. 카테고리 매칭         │
│ 3. 신뢰도 계산           │
└────────┬────────────────┘
         │
         ▼
    ┌────┴────┐
    │ 신뢰도? │
    └────┬────┘
         │
    ┌────┴────┐
    │         │
 >= 0.7    < 0.7
    │         │
    ▼         ▼
┌────────┐ ┌──────────────┐
│명확     │ │신뢰도 낮음   │
│분류    │ │→ 미분류      │
└───┬────┘ └──────────────┘
    │
    ▼
┌─────────────────────┐
│ 시스템 조치 필요?    │
└────────┬────────────┘
         │
    ┌────┴────┐
    │         │
   Yes       No
    │         │
    ▼         ▼
┌────────┐ ┌────────────┐
│답변안함│ │답변 생성    │
│운영자  │ │(Prompt/RAG)│
└────────┘ └────────────┘
```

## 4. 신뢰도 레벨 (Confidence Level)

### 신뢰도 레벨 정의

| 레벨 | 한글명 | 조치 |
|--------|------|------|
| very_high | 매우 높음 | 자동 답변 (검토 불필요) |
| high | 높음 | 자동 답변 (선택적 검토) |
| medium | 중간 | **운영자 검토 필수** |
| low | 낮음 | **미분류 → 운영자** |

### 신뢰도 레벨 결정 요인

LLM이 다음 4가지 요소를 종합적으로 고려하여 레벨 선택:

1. **키워드 명확성**
   - 명확한 키워드 → 높은 신뢰도
   - 예: "과제 제출" → very_high

2. **문의 구체성**
   - 구체적 → 높은 신뢰도
   - 예: "TypeError 발생" → high

3. **카테고리 적합도**
   - 하나의 카테고리에 명확히 속함 → 높은 신뢰도
   - 여러 카테고리 경계선 → 낮은 신뢰도

4. **과거 사례 유사성**
   - 유사 사례 많음 → 높은 신뢰도
   - 전례 없음 → 낮은 신뢰도

### 각 레벨별 판단 기준

#### very_high (매우 높음)
- 매우 명확한 키워드 사용
- 구체적이고 명확한 문의
- 하나의 카테고리에 완벽히 매칭
- 과거 사례가 많을 것으로 예상

#### high (높음)
- 명확한 키워드 존재
- 구체적인 문의
- 명확한 카테고리 분류 가능
- 과거 사례 존재 가능성 높음

#### medium (중간)
- 일부 키워드 모호
- 문의 내용 일부 불명확
- 카테고리 경계선상
- 과거 사례 적을 가능성

#### low (낮음)
- 키워드 불명확 또는 없음
- 문의 내용 매우 불명확
- 여러 주제 혼재
- 카테고리 분류 어려움

## 5. 보수적 답변 전략

### 원칙: "확실할 때만 답변, 의심스러우면 사람에게"

```python
# 의사 결정 트리
if confidence_level == "low":
    → 미분류 (운영자)

elif system_action_required:
    → 답변 안함 (운영자)

elif confidence_level == "medium":
    → 답변 가능하지만 운영자 검토 필요

else:  # very_high or high
    → 자동 답변
```

### 보수적 판단이 필요한 경우

1. **개인정보 관련**
   - 신뢰도 레벨 관계없이 무조건 운영자

2. **시스템 장애**
   - 신뢰도 레벨 관계없이 무조건 운영자

3. **정책/규정 변경**
   - 신뢰도 레벨 very_high만 자동 답변

4. **복잡한 코드 에러**
   - 신뢰도 레벨 high 이상 + 과거 사례 있을 때만

## 6. 실제 예시

### 예시 1: 과제 제출 규정

**문의**: "최종과제는 여러 번 제출 가능한가요?"

**LLM 판단**:
```json
{
  "label": "SUBMISSION_POLICY",
  "should_respond": true,
  "strategy": "prompt_direct",
  "confidence_level": "very_high",
  "rationale": "명확한 제출 규정 문의, FAQ로 즉시 답변 가능"
}
```

**이유**:
- ✅ 명확한 분류 (very_high)
- ✅ FAQ 답변 가능
- ✅ 시스템 조치 불필요

---

### 예시 2: 인증 버튼 비활성화

**문의**: "인증 버튼이 비활성화되어 있습니다"

**LLM 판단**:
```json
{
  "label": "ACCOUNT_ACTION_REQUIRED",
  "should_respond": false,
  "strategy": "no_response",
  "confidence_level": "high",
  "rationale": "개인 계정 DB 직접 조작 필요, Agent 권한 없음"
}
```

**이유**:
- ✅ 명확한 분류 (high)
- ❌ 계정 DB 조작 필요
- ❌ Agent가 처리 불가능

---

### 예시 3: 애매한 문의

**문의**: "안돼요"

**LLM 판단**:
```json
{
  "label": "UNCATEGORIZED",
  "should_respond": false,
  "strategy": "no_response",
  "confidence_level": "low",
  "rationale": "문의 내용 불명확, 추가 정보 필요"
}
```

**이유**:
- ❌ 낮은 신뢰도 (low)
- ❌ 무엇이 안되는지 불명확
- → 미분류

---

### 예시 4: 복합 문의

**문의**: "강의를 듣고 싶은데 과제도 제출해야 하나요? 그리고 인증도 받고 싶습니다"

**LLM 판단**:
```json
{
  "label": "UNCATEGORIZED",
  "should_respond": false,
  "strategy": "no_response",
  "confidence_level": "low",
  "rationale": "여러 주제 혼재 (강의+과제+인증), 명확한 분류 어려움"
}
```

**이유**:
- ❌ 낮은 신뢰도 (low)
- ❌ 3개 주제 혼재
- → 미분류 (운영자가 분리해서 답변)

## 8. 요약

### 답변 생성 조건
```
신뢰도 레벨: very_high OR high
AND
(FAQ 가능 OR RAG+Tool 가능)
AND
시스템 조치 불필요
```

### 미분류 조건
```
신뢰도 레벨: low
OR
카테고리 없음
OR
여러 주제 혼재
OR
내용 불명확
```

### 신뢰도 레벨별 조치
- **very_high**: 자동 답변 (검토 불필요)
- **high**: 자동 답변 (선택적 검토)
- **medium**: 운영자 검토 필수
- **low**: 미분류 → 운영자

### 보수적 전략
**"확실하지 않으면 운영자에게"**
